Q&A

> ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

La validación del lado del cliente es una verificación inicial que mejora tanto la experiencia del usuario como la respuesta del sistema, pues ayuda a garantizar que los datos enviados coinciden con los requisitos establecidos y, en caso de no hacerlo, solicita una corrección inmediata (proceso que tardaría más si la alerta proviene directamente del servidor)

> ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Deshabilitando javascript en el navegador o mediante inyección de código

Referencia 1

> Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Porque si el usuario no es malicioso, pero cometió algún error al ingresar su información, el alertar el error inmediatamente mejora su experiencia (pues no tiene que esperar a que el servidor rechace sus datos); además, este tipo de validaciones reducen la carga de procesamiento del servidor
